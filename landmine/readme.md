# Landmine Game (Realtime Multiplayer)

多人数リアルタイム通信を用いた地雷ゲーム。

## 設計方針
- **Lobby:** Firebase Realtime Database を使用し、ホストの動的委譲と参加者同期を行う。
- **Game:** `landmine.html` にてゲームロジックを実行。ホストが開始ボタンを押すことで全員の画面が同期する。
- **Security:** Firebase Database Rules にて、ルーム参加者のみがデータを読み書きできるように制限。

## 開発運用ルール
- ログ出力形式（history.json・README更新テーブル）を遵守すること。



## 技術的要点と注意点

### ロビー機能およびホスト委譲ロジックについて
ロビーおよびホスト委譲の実装において、以下の点に注意して設計を行っています。

*   **`onDisconnect` の制限と回避:** 
    `onDisconnect` は `set` や `remove` などの単純な操作のみをサポートしています。`transaction` を直接メソッドとして呼び出すことはできないため、離脱時の状態更新は「単純な削除予約」と、残ったメンバーによる `onValue` 監視・立候補という「検知→立候補」の2段階構成で実装しています。
*   **非同期競合の回避:**
    Firebaseのキャッシュとサーバー側の状態不一致を防ぐため、`snapshot.exists()` による存在チェックと、データが `null` の場合の強制リダイレクト処理を徹底し、セッションの不整合を回避しています。
*   **名前情報の永続化:** 
    ホストが部屋作成後にページ遷移が発生しても、`sessionStorage` を用いて名前情報を保持することで、ユーザーに再入力を求めないスムーズな参加フローを実現しています。
*   **ホスト選出の堅牢性:** 
    特定のホストを維持するのではなく、「ホストIDが空である」という状態を全員で監視し、空であれば即座に立候補（`runTransaction`）する「立候補制」を採用しました。これにより、ホストの離脱理由（通信切断・ブラウザ終了・リロード等）に関わらず、確実に新しいホストが選出されます。
